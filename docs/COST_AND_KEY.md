# עלויות, מפתח API, ואסטרטגיות הוזלה — הסבר מלא

> מסמך הסבר. נוגע במושגי-יסוד של תמחור LLM ובהחלטות אסטרטגיות למוצר.

## 1. מפתח API — מה זה ולמה צריך
**API** = ממשק שדרכו האפליקציה שלנו "מדברת" עם Claude. יצירת ערך = בקשת API.
**מפתח API** (`sk-ant-...`) = סיסמה סודית שמזהה מי מבצע את הבקשה ו**מחייבת את חשבון Anthropic שלך**.
המפתח של האפליקציה **נפרד לגמרי** מהקרדיטים של סשן הקוד (Claude Code) — שני חשבונות, שני מונים.

## 2. טוקנים — יחידת התמחור
**טוקן** = פיסת טקסט (~3-4 תווים באנגלית). המודל סופר טוקנים, לא מילים.
**עברית יקרה יותר** — הטוקנייזר אומן בעיקר על אנגלית, אז עברית מתפרקת ל~1.3-2× טוקנים למילה.
מחייבים בנפרד על **קלט** (מה ששולחים) ו**פלט** (מה שהמודל מייצר, כולל "חשיבה").

| מודל | קלט /1M | פלט /1M |
|---|---|---|
| Opus 4.8 | $5 | $25 |
| Sonnet 5 | $3 | $15 |
| Haiku 4.5 | $1 | $5 |

פלט יקר פי-5 מקלט. חשיבה (adaptive thinking) נספרת כפלט.

## 3. עלות ערך אחד
```
קלט ~1,500 טוקנים  × $5/1M   = $0.0075
פלט ~6,000 טוקנים  × $25/1M  = $0.15
סה"כ ≈ $0.16  (טווח מציאותי $0.15–0.40)
```

## 4. Cache — המושג שמשנה הכול
**Cache (מטמון)** = לשמור תוצאה שכבר חושבה. ערך מיוצר **פעם אחת**, נשמר במסד הנתונים, ומוגש
בחינם לכל הקוראים לנצח. **העלות = מספר הנושאים הייחודיים, לא מספר הצפיות.** כמו ויקיפדיה.

## 5. הוזלה בקנה-מידה גדול
השאלה "30,000 ערכים = אלפי דולרים" — התשובה: **לא מייצרים 30,000 מראש.**

- **יצירה עצלה (lazy / on-demand):** המנוף הגדול ביותר. ערך נוצר **רק כשמישהו באמת מבקש אותו**.
  הביקוש הוא "זנב ארוך" — כמה מאות נושאים פופולריים מכסים את רוב התנועה. משלמים על ביקוש אמיתי, לא על מלאי.
- **Batch API:** יצירה בכמות (לא בזמן אמת) ב-**50% הנחה**.
- **Prompt caching:** הנחיית המערכת (החוקה) זהה בכל יצירה → אפשר לשמור אותה ולשלם ~90% פחות על החלק הזה.
- **דירוג מודלים (tiering):** "נתב" שבוחר מודל לפי קושי הנושא — Opus לנושאים רגישים/שנויים, Sonnet/Haiku
  לפשוטים. יכול לחצות את העלות הממוצעת. eval-gated.
- **כיוונון effort ואורך פלט:** פחות חשיבה / ערכים תמציתיים = פחות טוקני פלט (הצד היקר).
- **הצעד ההיברידי:** בטווח ארוך בני אדם עורכים/מוסיפים → פחות תלות ב-LLM.

**המסקנה:** עם יצירה עצלה + caching + batch + tiering, העלות **פרופורציונלית לשימוש אמיתי**, לא קיר של אלפי דולרים מראש.

## 6. סימביוזה בין כמה LLM לביטול הטיות
**Ensemble רב-מודלי** = להשתמש בכמה מודלים עם הטיות אימון *שונות*, כך שההטיות מבטלות זו את זו.
- **ביקורת חוצת-מודלים (cross-model audit) ⭐:** מודל אחד מייצר, מודל **אחר** (משפחה אחרת — GPT/Gemini)
  בודק הטיה. זה בדיוק פותר את בעיית "נקודות העיוורון המשותפות" של מבקר מאותה משפחה. **מומלץ ל-v2.**
- **יצירה רב-מודלית:** לייצר כל עדשה בכמה מודלים ולמזג. **יקר** (עלות מוכפלת) ופוגע בעקביות הסגנון.
- **דיבייט אדוורסרי:** שני מודלים מבקרים זה את זה.

**המתח (חשוב להבין):** רב-מודליות **מפחיתה הטיה אבל מייקרת** — כל מודל נוסף = עלות נוספת. אין ארוחת חינם.
המבקר-המסומא שבנינו (מודל אחד) הוא פתרון 80% חסכוני; ביקורת חוצת-מודלים היא שדרוג פרימיום.
**המלצה:** להשאיר יצירה ב-Opus (עקביות + איכות), ולשדרג את *המבקר* למודל אחר ב-v2 — עלות קטנה, רווח דה-ביאס גדול.

## 7. לייצר באנגלית ולתרגם?
מפתה (אנגלית זולה יותר בטוקנים, מודלים "חכמים" יותר בה), **אבל לא מתאים למוצר הזה:**
- **תרגום גם עולה** — Google Translate/DeepL (~$20/מיליון תווים) או תרגום ב-LLM (עוד קריאה) — מקזז את החיסכון.
- **אובדן נאמנות** — הליבה של perspectipedia היא ייצוג *מדויק, מכובד, תרבותית-מדויק*, הרבה ממנו מעוגן
  ב**מקורות עבריים** (פסוקים, מסכתות, מפרשים). יצירה באנגלית + תרגום:
  - מרסקת מונחים וציטוטים עבריים, משטחת את הרובד והרגיסטר של השיח הדתי העברי.
  - התוצאה קוראת כ"מתורגמת", לא אותנטית — פגיעה בדיוק בדבר שנותן ערך.
  - מוסיפה **שכבת עיוות שנייה** (הטיות ושגיאות של מנוע התרגום) על העיוות של המודל.
- **אין רווח אמיתי:** המודל ממילא "חושב" באיזו שפה שהוא רוצה פנימית — כפיית קלט/פלט אנגלי לא מוסיפה.
- עברית = ~1.3-2× טוקנים, מנוף עלות **מתון** — לא הדומיננטי (הדומיננטי הוא *כמה* ערכים מייצרים בכלל).

**מסקנה:** לייצר **native בעברית** עם Opus (חזק בעברית), ולחסוך במנופים של §5 — לא בתרגום.
