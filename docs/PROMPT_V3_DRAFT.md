## שינוי 1 — החלפת "עקרון-על 1" (טקסט מוכן-להדבקה)

מחליף את הבלוק הנוכחי "עקרון-על 1 — מבחן הביסוס" בשלמותו:

```
עקרון-על 1 — מבחן הביסוס הדו-מסלולי:
עדשה נכנסת לערך רק אם היא מבוססת. הרף נקבע לפי מה שהעדשה *טוענת* — לא לפי מי שמאחוריה.

מסלול א' — עדשות של דרך-ראייה (epistemic_type: meaning-narrative או value-position):
עדשה שמציעה מסגור, משמעות, ערכים או דרך לחיות — נכנסת אם היא דרך-ראייה מנוסחת
וקוהרנטית שקיימת בעולם: מסורת (גם שבעל-פה), הוגה, יצירה, אסכולה, חוכמה עממית, או
היגיון פנימי קוהרנטי. לא נדרש מוסד, לא אקדמיה, לא קונצנזוס ולא מקור כתוב. הביסוס
במסלול הזה = להראות מאיפה דרך-הראייה באה ואיך היא מחזיקה את עצמה — לא להוכיח שהיא
"נכונה". דרכי-ראייה שהמיינסטרים ממעט לייצג (חוכמת קומון-סנס, מסורות מחשבה לא-אקדמיות,
הוגים מחוץ לקאנון) שייכות כאן במלואן — אל תשמיט אותן ואל תסמן אותן נחותות.

מסלול ב' — טענות עובדתיות-מדידות (epistemic_type: empirical-grounded או
methodological-critique, וכן כל טענה מדידה המופיעה בתוך עדשה כלשהי, גם עדשת מסלול א'):
נדרשות ראיות ממשיות. טענה שסותרת קונצנזוס אמפירי מבוסס נכנסת אך ורק כביקורת
מתודולוגית שמתקיימת בתוך השיח המקצועי עצמו (מתפרסמת, נדונה, נענית) — לא כהכחשה.
קוהרנטיות פנימית איננה ביסוס במסלול ב'.

שאלת-הסיווג (הפעל אותה על כל עדשה מועמדת):
"אם אמחק מהעדשה כל טענה עובדתית-מדידה — האם נשארת בה דרך-ראייה עם תוכן?"
- נשארת → העדשה נכנסת במסלול א' על החלק הזה; כל טענה מדידה שבתוכה נבחנת בנפרד
  במסלול ב', ומה שלא עובר — לא ייכתב, בלי שהעדשה כולה תיפסל.
- לא נשארת (הליבה כולה טענות מדידות ללא ביסוס מסלול ב') → העדשה כולה בחוץ.

יישום ההבחנה בכתיבה: עדשת מסלול א' מדברת בשפת הנרטיב שלה ("כך מסורתנו מבינה את
ראשית הקיום") ואינה מנסחת את עצמה כטענה מדעית-מדידה או כהפרכת מדידה ("המדידה שגויה"
היא טענת מסלול ב'). אין בכך שום נחיתות — זו פשוט הבחנה בסוג הטענה, והערך מציג את
העדשות זו לצד זו בלי הכרעה. סמן epistemic_type לפי המסלול בפועל: לעולם אל תסמן
עדשת דרך-ראייה כ-empirical-grounded ולהפך.

לכל עדשה חובה למלא grounding אמיתי, ספציפי ומדויק — אל תמציא מקורות ואל תשים בפי
מסורת דברים שהיא לא באמת אומרת.
```

**הערות מנסח:** (א) הפסקה "דרכי-ראייה שהמיינסטרים ממעט לייצג" היא מימוש ישיר של הכרעת
המייסד — היא הופכת את הפתיחות מהיתר לחובה (מכריעה את מקרים 1, 2, 9 לכיוון recall).
(ב) המשפט האחרון נשמר מ-v2 בכוונה — רציפות עם "דיוק מקורות".

```
- crux (מוקד המחלוקת): אחרי כתיבת כל העדשות, נסח ב-2-4 משפטים: על מה בדיוק העדשות
```

### הרחבת סכמה (טיוטה, `src/lib/schema.ts` — `ENTRY_JSON_SCHEMA`)


שנוכחותו חייבת להיות מובטחת. צד ה-DB (`Entry.crux String?`) כבר קיים מהמיגרציה.

## שינוי 4 — שדה `url` אופציונלי בפריטי grounding

תוספת לבלוק "דיוק מקורות (קריטי)", אחרי "עדיף מקור כללי-ונכון על פני ציטוט מדויק-ומומצא":

```
כאשר למקור יש עותק דיגיטלי מוכר וקנוני — ספק url: ספריא (sefaria.org.il) למקורות
יהודיים, ויקיטקסט לטקסטים עבריים, פרויקט בן-יהודה לספרות עברית, DOI/PubMed למחקרים.
כללי הזהב: (1) ספק url רק אם אתה בטוח בו — url מומצא או משוער גרוע בהרבה מהיעדר url;
(2) היעדר url לגיטימי לחלוטין, במיוחד לעדשות מסלול א' — מסורת שבעל-פה, חוכמה עממית
או הוגה שאין לו טקסט מקוון הם ביסוס מלא גם בלי קישור, ואל "תשדרג" אותם במקור כתוב
מומצא; (3) לטענות מסלול ב' (empirical-grounded / methodological-critique) שאף על
מחקר או פרסום — השתדל לספק url או מזהה (DOI), כי שם הביסוס אמור להיות בר-אחזור.
```

### הרחבת סכמה (טיוטה — בתוך items של `grounding`)

```jsonc

  "type": "string",
  "description":
    "אופציונלי: קישור לעותק דיגיטלי קנוני של המקור (ספריא/ויקיטקסט/בן-יהודה/DOI). רק אם ודאי. היעדרו לגיטימי."
}
// "required" של פריט grounding נשאר ["source", "explanation"] — url לא נכנס אליו.
```

(צד ה-DB: `grounding Json` כבר סופג את השדה בלי מיגרציה; ה-UI — badges "מקור מקושר" /

## שינוי 5 — חידוד מדיניות אדם חי + כנות בסירוב

מחליף את הבלוק "מדיניות תוכן (מנגנון סירוב)" בשלמותו:

```
מדיניות תוכן (מנגנון סירוב):
מבחן הביסוס הוא מסנן אפיסטמי, לא מוסרי — "מבוסס" אינו "מקובל". לכן, אם הנושא נופל
לאחד מאלה, החזר refused=true עם refusal_reason מכובד, ומערך lenses ריק:
- נושא שכל תכליתו הסתה, שנאה או הכפשה של קבוצה — וכן עדשה שליבתה שלילת זכויות של
  קבוצת בני אדם: גם אם יש לה מקורות ואסכולה, היא אינה מתארחת כאן. אם זו הסיבה —
  אמור זאת בכנות ב-refusal_reason: הסירוב הוא ממדיניות התוכן, לא מ"חוסר ביסוס".
- כל אדם חי מזוהה — פרטי או ציבורי — סירוב קטגורי ב-v1: לא ערך עליו, לא עדשות על
  אישיותו, מעשיו או משנתו. (דמויות היסטוריות שנפטרו — מותר, בזהירות: בלי האשמות
  שאינן קונצנזוס היסטורי מתועד.)
- תוכן מזיק אופרטיבית (הוראות לפגיעה, נשק, סמים וכו').
בכל שאר המקרים החזר refused=false ובנה ערך כרגיל. שים לב: נושא *על* תופעה בעייתית
("מדוע תיאוריות קשר משגשגות") הוא לגיטימי — החסימה היא על התופעה כעדשה, לא כמושא מחקר.
```

**מה השתנה מ-v2:** (א) "אדם פרטי חי" → "כל אדם חי מזוהה, פרטי או ציבורי" — קו חד וניתן
לאכיפה, עקבי עם שער-Haiku (PLAN 1.2) שחוסם עוד לפני Opus; קו-משנה מפורש לנפטרים.
(ב) חובת הכנות בנימוק הסירוב (מקרה 6 — כהנא: מדיניות, לא אפיסטמולוגיה). (ג) הבחנת
מושא/עדשה (מקרה 5).

### הרחבת סכמה נלווית (תיאור בלבד, לא מבנה)

עדכון ה-`description` של `refused` כך שישקף את הקו החדש:

```jsonc
"refused": {
  "type": "boolean",
  "description":
    "true אם הנושא מחוץ לתחום: הסתה/שנאה/שלילת זכויות קבוצה, כל אדם חי מזוהה (פרטי או ציבורי), או תוכן מזיק אופרטיבית. אחרת false."
}
```

## שינוי 6 — עומק ויקיפדי: `body` → `sections` (D6)

### 6א. תוספת לבלוק "כללי כתיבה" (מחליף את השורה "summary: 1-2 משפטים. body: 2-4 פסקאות.")

```
- summary: 1-2 משפטים.
- כל עדשה נכתבת בפרקים (sections): 4-6 פרקים, ובסך הכול 400-700 מילים לעדשה.
  אוצר הפרקים (בסדר הזה; התאם לעדשה, אל תמלא מכנית):
  1. "מבוא" (חובה) — הצגת דרך-הראייה ביחס לנושא.
  2. "יסודות דרך-הראייה" (חובה) — המושגים והטיעונים המרכזיים.
  3. "המקורות המרכזיים" — הרחבה נרטיבית של הביסוס: מה המקורות אומרים ואיך, לא רשימה.
  4. "התפתחות והקשר היסטורי".
  5. "זרמים ומחלוקות פנימיות" (חובה) — עולם-תוכן בלי מחלוקות פנימיות הוא קריקטורה;
     הצג את הפלורליזם הפנימי האמיתי.
  6. "ביקורות ותשובות" — הביקורות המרכזיות על דרך-הראייה הזו, והתשובות להן *בקולה
     של העדשה* — בכבוד לביקורות, בלי איש-קש.
  7. "משמעות מעשית" — אופציונלי.
- סימטריית מבנה: מספר הפרקים, אורכם ורמת הקונקרטיות דומים בין כל העדשות באותו ערך
  (הרחבה של עקרון-על 5.2 — שוויון עומק חל גם על המבנה).
```

### 6ב. הרחבת סכמה (טיוטה — בתוך items של `lenses`)

```jsonc
// מחליף את "body" בפלט המודל:
"sections": {
  "type": "array",
  "minItems": 3,
  "maxItems": 7,
  "description": "פרקי העדשה, לפי אוצר הפרקים שבהנחיה. 400-700 מילים בסך הכול.",
  "items": {
    "type": "object",
    "required": ["heading", "content"],
    "additionalProperties": false,
    "properties": {
      "heading": { "type": "string", "description": "כותרת הפרק (מאוצר הפרקים או קרובה אליו)" },
      "content": { "type": "string", "description": "תוכן הפרק — פסקה או יותר" }
    }
  }
}
// "required" של עדשה: ["name","family","summary","sections","grounding","epistemic_type"]
// (body יוצא מהסכמה — ראו 6ג לתאימות לאחור בצד ה-DB.)
```

### 6ג. צד הקוד בנחיתה (לא בסכמת המודל)

- ‏Prisma: עמודת `sections Json?` חדשה על Lens; **`body` נשאר** ומאוכלס בשרשור הפרקים
  (`heading + "\n" + content`) — תאימות לערכי v2/seed, `passesGroundingBar` ממשיך לבדוק
  אורך על שדה אחד (ולהעלות את הסף: ≥1,500 תווים לעדשת v3), וחיפוש עתידי על טקסט רציף.
- ‏`claude.ts`: ‏`max_tokens` ‏16,000 → **32,000**.
- הרנדרר מעדיף `sections` כשקיים (PLAN 4.5); ערכי v2 מרונדרים מ-body כרגיל.
- ‏`Entry.depth` (`standard|deep`, ברירת מחדל standard) — עמודה + פרמטר יצירה (PLAN 4.7);
  ב-v3 נתמך רק standard, deep נכנס עם הפיצול ב-v3.5 (D6).

| 4. אנטי-חיסונים | ⛔ + עדשה ערכית כן | שאלת-הסיווג (ליבה מדידה → בחוץ); העדשה הערכית = value-position מסלול א' |

| 6. כהנאות | ⛔ במדיניות, בכנות | סעיף "שלילת זכויות של קבוצה" + חובת הכנות ב-refusal_reason |
| 7. רפואה משלימה | ◐ | שאלת-הסיווג + כלל ה-confidence הקיים (עקרון 2/כללי כתיבה — ללא שינוי) |
| 8. אקלים | ◐ ביקורת מקצועית כן, הכחשה לא | מסלול ב': "מתקיימת בתוך השיח המקצועי עצמו" + עקרון-על 4 |
| 9. מסורת שבעל-פה | ✅ בלי url | מסלול א' ("גם שבעל-פה... לא מקור כתוב") + כלל זהב 2 בשינוי 4 (איסור "שדרוג" מומצא) |
| 10. בריאה מילולית | ◐ נרטיב כן, אנטי-דאטציה לא | "יישום ההבחנה בכתיבה" (שפת הנרטיב; "המדידה שגויה" = מסלול ב') |
