# perspectipedia

**אנציקלופדיה הוגנת, מרובת נקודות מבט.** קראו כל נושא דרך כמה עדשות מבוססות — בכבוד — והשוו ביניהן.

> לא "מי צודק", אלא איך כל עולם מבין את זה — ועל מה הוא מבסס.

## הרעיון

מקורות הידע הקיימים מתיימרים ל"אמת אחת ניטרלית". על שאלות של משמעות, מוצא, ערכים ופרשנות —
יש כמה נרטיבים לגיטימיים. perspectipedia מציגה את אותו נושא דרך כמה עדשות (יהודית, מדעית,
פילוסופית, ...), כשכל אחת עומדת בכבוד בפני עצמה, והקורא בוחר ומשווה.

**עיקרון-על — מבחן הביסוס:** עדשה נכנסת רק אם היא מנומקת ומבוססת. כל עדשה מוצגת עם
"על מה זה מבוסס" (מקורות, אסכולות, ראיות). עמדות חסרות ביסוס אינן נכללות.

ראו את המסמכים המלאים ב-[`docs/`](./docs): אפיון מוצר וארכיטקטורה.

## סטאק

- **Next.js** (App Router) + **TypeScript** + **Tailwind CSS** (RTL, עברית)
- **Prisma** + **SQLite** — שמירת ערכים (הזרע ל"ויקיפדיה")
- **Claude Opus 4.8** (`@anthropic-ai/sdk`) — מנוע יצירת העדשות, עם פלט מובנה שאוכף את מבחן הביסוס

## הפעלה מקומית

```bash
# 1. התקנת תלויות
npm install

# 2. הגדרת מפתח API — העתיקו את .env.example ל-.env.local ומלאו:
#    ANTHROPIC_API_KEY=sk-ant-...
cp .env.example .env.local

# 3. יצירת מסד הנתונים
npm run db:push

# 4. הרצה
npm run dev
```

פתחו http://localhost:3000, הקלידו נושא, וקבלו אותו דרך כמה עדשות מבוססות.

> **הערה:** בלי `ANTHROPIC_API_KEY` האתר יעלה, אבל יצירת ערכים חדשים תיכשל עם הודעה מתאימה.

## מבנה

```
src/
├── app/            # דפים ו-API (Next.js App Router)
│   ├── page.tsx            # דף בית
│   ├── entry/[slug]/       # דף ערך
│   └── api/entry/          # יצירה/שליפה של ערך
├── lib/
│   ├── claude.ts   # מנוע ה-LLM + אכיפת מבחן הביסוס
│   ├── prompts.ts  # הנחיית המערכת (4 עקרונות-העל)
│   ├── schema.ts   # JSON Schema (מבחן הביסוס בקוד)
│   └── entryService.ts     # לולאת הליבה: cache → generate → persist
└── components/     # SearchBox, LensSwitcher, LensView, CompareView, ...
```

## סטטוס

גרסה 1 (MVP): לולאת הליבה — נושא → עדשות → ביסוס → השוואה → **שמירה**.
בלי חשבונות משתמש ובלי עריכה קהילתית (מתוכננים לגרסאות הבאות; ראו מפת הדרכים באפיון).
